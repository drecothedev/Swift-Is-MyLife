Introduction to Language building in Swift! 

Ciao!👋🏽 This is a very important topic to me personally. Even though i use Swift, recently i haven't built any apps. I started learning Swift to learn how to make apps but after countless practice apps, i needed something a bit more stimulating. I landed on compilers. Programming languages work like magic to me. They work every single time, unless of course of i ( the programmer/user ) did something wrong. This expected behavior is persistent through different languages with design patterns, compiling methods, syntaxes, etc. 

Building a langauge also feels like magic. In many ways building a programming languages makes you think about programming in very different ways. Instead of scavening to find out how to do something you have to think very critically about how you to set the correct rules for acheiving a certain behaviour. These rules must be precise as the behavior needs to be consistent. This will sound hippieish but *bleep* it. To me, building a langauge feels like builiding a little city or civilization ( sorry i am a huge history fan too, so this makes it sound fun to me ). How? becasue you have to assign/manage behaviour, prescendence ( level of importance ) , rules ( laws ), trade offs ( utilitarianism ), garbage collection ( debt ), etc. So, i know "building a little world" seems a bit like an exaggeration, but you feel me. 


Throughout this directory i will be building a language from scratch. I will try my best to explain what the code is doing and write code that is ultra readable with helpful comments. Initially, i will be building just an interpreter and ( hopefully ) adding some actual compiling VM type shizz. I say hopefully because i have not yet mastered virtual ( and frankly i haven't mastered interpreters neither, but i have at least written one ).
